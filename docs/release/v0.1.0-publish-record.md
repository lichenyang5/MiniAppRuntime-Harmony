## Security Gate

`build-profile.json5` 中的签名凭据已经进入公开远端历史。必须先完成以下事项，才能继续 tag 或 Release：

1. 轮换或吊销相关签名凭据。
2. 从当前工程移除签名材料，改用本地且不入库的配置。
3. 评估并清理 Git 历史中的敏感内容。
4. 推送清理结果后重新确认仓库内容和 CI。

在安全门禁完成前，本文档中的发布命令只用于说明，不应执行。

## What Is Included

- ArkWeb 本地 H5 容器、加载状态、URL Guard 和错误状态页。
- H5 SDK 的 requestId、callback map、Promise、timeout 和 CALLBACK_LOST。
- JavaScriptProxy、BridgeController、Dispatcher、Registry、Bootstrap。
- Biz/Imp 与 BridgeCallbackExecutor。
- Toast、Clipboard、Storage、`runtime.getApiList` 共 8 个 action。
- ApiManifest、生成 API 文档、typed API 和 DebugPanel。
- `myascf_runtime` 本地 HAR 与 IIFE/ESM H5 SDK。
- Node tests、一致性检查、npm pack dry-run、CI 与手工 smoke test 文档。

## What Is Not Included

- npm 未发布，H5 SDK 仍为 `private: true`。
- ohpm 未发布，HAR 当前仍通过本地 file dependency 使用。
- CLI、Network API、Device API 未实现。
- 不包含生产级安全沙箱、完整设备兼容矩阵或自动化真机验证。

## Manual Publish Instructions

完成 Security Gate、手工回归和最终 CI 后，由项目作者逐条确认并手动执行：

```bash
git status
git add .
git commit -m "docs: prepare v0.1.0 showcase release"
git push origin main
```

等待该提交的 GitHub Actions 成功后，再执行：

```bash
git tag -a v0.1.0 -m "MiniAppRuntime-Harmony v0.1.0"
git push origin v0.1.0
```

最后在 GitHub Releases 页面选择 `v0.1.0`，粘贴 [GitHub Release Template](github-release-template.md)，补齐真实环境和链接后手动发布。如果实际主分支发生变化，应将命令中的 `main` 调整为真实分支。
