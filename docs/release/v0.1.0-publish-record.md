# v0.1.0 npm 发布与消费验证记录

这篇文档解决什么问题：记录 H5 SDK `v0.1.0` 的真实 npm 状态，以及 npm SDK 与本地 HAR 在独立 HarmonyOS Demo 中的验证边界。

## npm 状态

```text
Package: @lcy453/miniapp-runtime-harmony-web-sdk
Version: 0.1.0
Dist tag: latest -> 0.1.0
Registry: https://registry.npmjs.org/
```

独立 React/Vite 消费工程的 lockfile 显示依赖解析到 npm registry 的 `0.1.0` tarball，不是 `file:`、`link:`、`workspace:` 或本地 tgz。

## Web 消费验证

- npm install：通过，审计结果为 0 个漏洞。
- ESM import：通过。
- `initMyASCF` 与 `createTypedApi`：TypeScript 编译通过。
- Vite 6.4.3：构建通过。
- rawfile 同步：通过，只复制 `dist`。
- 资源路径：`./assets/*`，适合 ArkWeb rawfile。
- 普通浏览器：没有 Native Proxy 时返回 `NATIVE_UNAVAILABLE`，属于预期。

## HAR 消费验证

框架模块使用 `harTasks`，模块类型为 `har`，公共入口为模块根 `Index.ets`。`assembleHar` 已完成 ArkTS 编译、HAR 打包和 packing check。

独立 HarmonyOS Demo 使用：

```json5
{
  "dependencies": {
    "myascf_runtime": "file:./libs/myascf_runtime.har"
  }
}
```

验证结果：

- HAR 源产物与消费副本 SHA-256 一致。
- ohpm 安装成功。
- `import { MyASCFRuntime } from 'myascf_runtime'` 编译成功。
- 不再注册外部源码模块。
- `CompileArkTS`、`PackageHap`、`PackingCheck` 通过。
- 生成未签名 HAP。

## 真机边界

设备可被 HDC 识别，但安装未签名 HAP 返回 `no signature file`。因此下列运行结果尚未确认：

- 四个 H5/Native 全局对象类型。
- `ui.showToast`。
- `runtime.getApiList`。
- `system.storage.setItem/getItem`。
- `network.request`。

下一步由项目作者在独立 Demo 中配置 DevEco 自动签名，再执行 Run 和 HiLog 取证。签名证书、密码和本机路径不得提交。

## 安全与范围

- 本次没有重新发布 npm。
- 没有发布 HAR 到 ohpm。
- 没有新增 CLI 或业务 API。
- 没有复制或提交签名材料。
