# MiniAppRuntime-Harmony v0.1.0 Release Notes

这篇文档解决什么问题：整理 v0.1.0 GitHub 展示候选版的真实能力、验证范围和已知限制，供未来创建 GitHub Release 时复核使用。

## Version Positioning

`v0.1.0` 是 GitHub 展示候选版，用于展示受小程序运行时架构启发的 HarmonyOS Web 容器与 JSBridge 工程实践。当前提供源码、本地 HAR、H5 SDK 构建产物、Demo 和文档，不表示 npm、ohpm 或 HarmonyOS 包仓库已经发布。

## Highlights

- ArkWeb 加载本地 H5，并维护 loading、success、error 与 blocked 状态。
- 基于 JavaScriptProxy 和 `runJavaScript` 的 H5/ArkTS 双向通信。
- requestId、callback map、Promise、TIMEOUT 与 CALLBACK_LOST 生命周期治理。
- BridgeController、BridgeDispatcher、HandlerRegistry 与 RuntimeBootstrap。
- Biz/Imp 分层和 BridgeCallbackExecutor 统一响应出口。
- Toast、Clipboard、Storage 与 `runtime.getApiList` 共 8 个内置 action。
- API Manifest、生成型 API 文档和 DebugPanel 动态能力展示。
- `myascf_runtime` 本地 HAR 与 `MyASCFRuntime` 门面。
- `h5_sdk` IIFE/ESM 双产物、TypeScript 声明、`sendTyped` 和 typed helper。
- Node 单元测试、Manifest/生成物/package 一致性检查与 GitHub Actions CI 配置。

## Validation Status

- 本地 `npm run check` 已建立为统一质量入口。
- H5 SDK 当前包含 13 个 Node 单元测试。
- npm pack dry-run、候选 tarball 和 external consumer 已完成本地预检。
- HAP 与设备功能仍以发布前 checklist 和手工 smoke test 的当次记录为准。
- GitHub Actions 需要在候选 commit 推送后确认真实线上结果。

## Not Included

- 尚未执行 npm publish 或 ohpm 发布。
- 尚未创建 v0.1.0 tag 或 GitHub Release。
- 不包含 Network、Device API 或完整 UI 组件 DSL。
- 不提供生产级安全沙箱、远程内容治理或完整兼容性承诺。
- 真实设备截图、权限行为和 CI 成功截图必须在实际验收后补充。

## Upgrade Notes

这是首个展示候选版本，没有历史版本迁移要求。现有 action 名、BridgeRequest/BridgeResponse 和 H5 全局边界保持当前文档约定。
